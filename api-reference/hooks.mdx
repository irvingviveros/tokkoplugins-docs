# Hooks de Acción

Los hooks de acción permiten ejecutar código personalizado en momentos específicos del ciclo de vida del plugin. Utiliza `add_action()` para conectarse a estos hooks.

## Eventos de sincronización (WP-Cron)

<Info>
Los siguientes nombres son **eventos WP-Cron** registrados por el plugin, no acciones lanzadas con `do_action()`. Puedes engancharte a ellos con `add_action()` para ejecutar código junto a la rutina del plugin.
</Info>

### thi_scheduled_sync

Evento programado que dispara la sincronización automática (solo planes premium).

<CodeGroup>
```php Ejemplo
add_action('thi_scheduled_sync', function() {
    error_log('Iniciando sincronización programada');
});
```
</CodeGroup>

**Parámetros**: Ninguno

**Descripción**: WP-Cron registra este evento con la frecuencia configurada en **Tokko Plugins → Configuración → Sincronización**. El plugin lo escucha para ejecutar `thi_premium_property_sync()`.

**Listener definido en**: `tokko-houzez-integration.php`

---

### thi_continue_image_sync

Evento por lotes (WP-Cron / Action Scheduler) que continúa la descarga de imágenes entre lotes.

<CodeGroup>
```php Ejemplo
add_action('thi_continue_image_sync', function() {
    update_option(
        'custom_image_batch_count',
        intval(get_option('custom_image_batch_count', 0)) + 1,
        false
    );
}, 20);
```
</CodeGroup>

**Descripción**: Se dispara repetidamente mientras queden imágenes pendientes por procesar, hasta que termine la sincronización.

**Listener definido en**: `includes/sync/sync-functions.php`

---

### thi_bulk_update_titles_batch

Evento por lotes para actualizar masivamente títulos de propiedades cuando cambia la configuración "Usar dirección como título".

<CodeGroup>
```php Ejemplo
add_action('thi_bulk_update_titles_batch', function() {
    error_log('Procesando lote de actualización de títulos');
}, 20);
```
</CodeGroup>

**Listener definido en**: `includes/sync/sync-functions.php`

---

### thi_cron_sync_continue

Evento de continuación de una sincronización cron que se pausó por timeout.

<CodeGroup>
```php Ejemplo
add_action('thi_cron_sync_continue', function() {
    error_log('Reanudando sincronización después de pausa');
}, 20);
```
</CodeGroup>

**Listener definido en**: `includes/sync/cron-handler.php`

## Hooks de integración

### tp_fs_loaded

Se dispara cuando el SDK de Freemius se ha cargado completamente.

<CodeGroup>
```php Hook
do_action('tp_fs_loaded');
```

```php Ejemplo
add_action('tp_fs_loaded', function() {
    // Acceso seguro a funcionalidad de Freemius
    if (function_exists('tp_fs')) {
        $user = tp_fs()->get_user();
        error_log('Usuario Freemius: ' . $user->email);
    }
});
```
</CodeGroup>

**Parámetros**: Ninguno

**Descripción**: Proporciona un punto seguro para inicializar código dependiente de Freemius después de que se ha cargado el SDK.

**Ubicación definida**: `tokko-houzez-integration.php`

## Hooks de AJAX

Estos hooks se registran vía `wp_ajax_{action}` y son llamados por la interfaz de administración del plugin. Todos los handlers viven en `includes/ajax/ajax-handlers.php`. Puedes engancharte **antes** (vía `add_action` con prioridad baja) o **después** (prioridad alta) para extender el comportamiento.

<Warning>
El plugin ya registra su propio handler para cada acción. Si agregas otro, asegúrate de usar una prioridad distinta a `10` para no reemplazar el comportamiento del plugin.
</Warning>

### thi_unified_sync_detect

Inicia la detección de cambios para sincronización.

<CodeGroup>
```php Hook
do_action('wp_ajax_thi_unified_sync_detect');
```

```php Ejemplo
add_action('wp_ajax_thi_unified_sync_detect', function() {
    // Validar capacidades de usuario
    if (!current_user_can('manage_options')) {
        wp_die('Acceso denegado');
    }
    // Código personalizado...
});
```
</CodeGroup>

**Descripción**: Punto de entrada para detección de cambios en sincronización AJAX.

---

### thi_unified_sync_execute_batch

Ejecuta un lote de sincronización.

<CodeGroup>
```php Hook
do_action('wp_ajax_thi_unified_sync_execute_batch', $batch_data);
```

```php Ejemplo
add_action('wp_ajax_thi_unified_sync_execute_batch', function() {
    // Procesar lote enviado
    $batch = isset($_POST['batch']) ? json_decode(wp_unslash($_POST['batch']), true) : [];
    error_log('Procesando lote de ' . count($batch) . ' elementos');
});
```
</CodeGroup>

**Descripción**: Se dispara para ejecutar un lote individual durante sincronización AJAX.

---

### thi_unified_sync_finalize

Finaliza la sincronización AJAX.

<CodeGroup>
```php Hook
do_action('wp_ajax_thi_unified_sync_finalize');
```

```php Ejemplo
add_action('wp_ajax_thi_unified_sync_finalize', function() {
    // Limpieza final
    delete_transient('thi_sync_in_progress');
    wp_cache_flush();
});
```
</CodeGroup>

**Descripción**: Se dispara al finalizar la sincronización AJAX.

---

### thi_sync_single_property

Sincroniza una propiedad individual.

<CodeGroup>
```php Hook
do_action('wp_ajax_thi_sync_single_property', $property_id);
```

```php Ejemplo
add_action('wp_ajax_thi_sync_single_property', function() {
    $prop_id = isset($_POST['property_id']) ? intval($_POST['property_id']) : 0;
    if ($prop_id > 0) {
        // Sincronizar propiedad individual
    }
});
```
</CodeGroup>

**Descripción**: Permite sincronizar una propiedad específica bajo demanda.

---

<CodeGroup>

add_action('wp_ajax_thi_sync_developments_only', function() {

---

### thi_force_reprocess_properties

Fuerza reprocesamiento de todas las propiedades.

<CodeGroup>
```php Hook
do_action('wp_ajax_thi_force_reprocess_properties');
```

```php Ejemplo
add_action('wp_ajax_thi_force_reprocess_properties', function() {
    // Limpiar caché de propiedades
    delete_transient('thi_cached_properties');
    // Iniciar reprocesamiento
});
```
</CodeGroup>

**Descripción**: Borra el caché de propiedades y reinicia el procesamiento desde cero.

## Mejores prácticas

<Note>
**Siempre valida capacidades de usuario** usando `current_user_can()` en hooks de administración.
</Note>

<Tip>
**Usa opciones transitorias** para datos temporales en hooks de sincronización para no sobrecargar la base de datos.
</Tip>

```php
// Buen ejemplo
add_action('thi_scheduled_sync', function() {
    // Almacenar datos temporales
    set_transient('thi_custom_sync_log', [
        'started' => current_time('mysql'),
        'status' => 'processing'
    ], 12 * HOUR_IN_SECONDS);

    // Ejecutar lógica personalizada
    // ...
});
```

## Performance

- Mantén hooks ligeros; no ejecutes operaciones pesadas
- Usa `wp_remote_post()` asincrónico para llamadas externas
- Aprovecha el sistema de caché de WordPress
